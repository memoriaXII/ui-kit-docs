## Default

`Layout` is the original full-screen app shell. Has built-in scroll state, loading screens, top nav, sticky footer. Showcased here inside a phone-frame mock so the demo reads as a screen.

<ComponentPreview name="layout-default" />

## With loading state

Pass `screenState={{ isLoading: true }}` + a `loadingNode={<LayoutLoading />}` for the initial render of a route.

<ComponentPreview name="layout-loading" />

## When to use

- `Layout` — original full-screen app shell. Owns viewport, has built-in scroll state.
- [`ResponsiveLayout`](/docs/components/responsive-layout) — newer responsive shell that adds a desktop layer.

## Installation

```bash
pnpm add @appboxo/ui-kit
```

## Types

`Layout`, `LayoutProps`, `NavBarProps`, `ScreenStateProps` are all exported. `combineHooksResultsToLayoutScreenState` helps merge multiple data-loading hooks into Layout's screen state.
